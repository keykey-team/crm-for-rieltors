# FREEMO R - Платформа для рієлторів

## Overview
Full-featured CRM for real estate agents built with Next.js 14, PostgreSQL, Prisma, NextAuth.

## Auth & Roles
- email/password via NextAuth.js, middleware-protected
- Test user: john@doe.com / johndoe123 (admin)
- 3 roles: agent, director, admin
- Role-based access: agents see only their own data (leads/deals/tasks), directors and admins see all
- lib/role-guard.ts: getSessionUser(), hasRole(), requireRole(), ownershipFilter()
- Role badge displayed in sidebar next to user name

## Custom Permissions (per-user section access)
- User.permissions field: JSON string array of allowed section keys, null = full access
- lib/permissions.ts: CRM_SECTIONS, parsePermissions(), hasPermission(), sectionFromPath()
- 12 controllable sections: dashboard, deals, leads, properties, tasks, calendar, chat, analytics, automations, templates, knowledge-base, activity-log
- settings, capabilities, pricing — always accessible (not restricted)
- Admin/director always have full access regardless of permissions field
- Agents with null permissions = full access (backwards compatible)
- Middleware enforces section access (redirects to /dashboard or /settings if denied)
- auth.ts passes permissions through JWT token → session
- Sidebar filters nav items based on permissions
- UI: Settings > Users tab shows permissions badge per user + PermissionsDialog with checkboxes
- API: /api/users GET/POST and /api/users/[id] PUT return and accept permissions field

## KeyKey Хелпер & AI Асистент
- Custom-built chat component with two modes (tabs): Довідка (Helper) and Дані CRM (Assistant)
- Uses Abacus AI LLM API (gpt-5.4-mini) via /api/helper and /api/assistant endpoints
- **Helper mode:** knowledge base from public/keykey-crm-guide.md (393 lines), streaming responses
- **Assistant mode:** two-step LLM pipeline:
  1. LLM parses user question → structured query params (entity, filters, etc.)
  2. lib/assistant-queries.ts executes precise DB query with ownershipFilter
  3. LLM generates natural language response from query results
  - Supports: leads, deals, properties, tasks, events, analytics, users queries
  - Respects role-based access (agents see only their data)
- Usage tracking: HelperMessage model in DB (userId, role, content, createdAt)
- Assistant messages prefixed with `[assistant]` in content for distinction
- Shared plan-based limits per calendar month:
  - Free: 20 requests/month
  - Pro: 200 requests/month
  - Business: 1000 requests/month
- lib/helper-limits.ts: HELPER_LIMITS, getHelperLimit(), getMonthStart()
- API: GET /api/helper returns usage stats + chat history + team stats (admin only)
- API: POST /api/helper sends helper message with streaming
- API: POST /api/assistant sends assistant query with streaming
- UI: components/helper-chat.tsx — floating button (bottom-right) + slide-up chat panel
- Mode switcher tabs in header, mode-specific welcome screens with example queries
- Styled in corporate colors (#073B34 button, #CEFD56 icon, usage progress bar)
- Click-outside-to-close behavior
- Chat history restored from DB on open (last 20 messages)
- Pricing page shows helper limits per plan
- Old iframe widget (App ID 790963704) removed, replaced with custom implementation
- Translation keys: helper.*, assistant.* (telegram.callTitle/callTooltip for Telegram bot button)
