# RealCRM - Платформа для рієлторів

## Overview
Full-featured CRM for real estate agents built with Next.js 14, PostgreSQL, Prisma, NextAuth.

## Design
- iOS-style, purple/violet primary (#7C3AED)
- DM Sans/Plus Jakarta Sans/JetBrains Mono fonts
- White cards with subtle shadows, rounded-xl/2xl corners
- Collapsible sidebar navigation

## Database Models
User, Lead, Property, PropertyPhoto, PropertyUnit, Deal, DealComment, DealChecklist, DealCustomField, DealCustomFieldValue, Task, Event, KnowledgeArticle, Automation, Template, ActivityLog, FunnelStage, Communication, AftercarePlan, AftercareStep, Dictionary, ChatMessage, LeadDistributionRule

## Pages (13 sidebar sections)
1. Dashboard - stats, recent leads, charts
2. Воронка (Deals) - Kanban board with dynamic stages from DB, deal detail with comments/checklist/history/custom fields
3. Ліди (Leads) - table/kanban views, lead detail with communication log, create deal from lead, Excel import
4. Об'єкти (Properties) - grid/list views, chess grid (shahmatka) for units
5. Задачі (Tasks) - CRUD with filters
6. Календар (Calendar) - event management
7. Аналітика (Analytics) - charts, agent activity, financials
8. Автоматизації (Automations) - trigger/action rules
9. Шаблони (Templates) - message templates
10. База знань (Knowledge Base) - articles by category
11. Чат (Chat) - internal messaging between employees
12. Налаштування (Settings) - 7 tabs: Profile (with avatar upload), Users (admin), Funnel stages, Custom fields, Dictionaries, Lead distribution rules, Aftercare plans

## Auth & Roles
- email/password via NextAuth.js, middleware-protected
- Test user: john@doe.com / johndoe123 (admin)
- 3 roles: agent, director, admin
- Role-based access: agents see only their own data (leads/deals/tasks), directors and admins see all
- lib/role-guard.ts: getSessionUser(), hasRole(), requireRole(), ownershipFilter()
- Role badge displayed in sidebar next to user name

## Localization (i18n)
- 3 languages: Ukrainian (uk), English (en), Russian (ru)
- lib/i18n/translations.ts: ~300 translation keys per locale
- lib/i18n/context.tsx: I18nProvider + useTranslation hook, persisted in localStorage
- Language switcher (UK/EN/RU buttons) in sidebar footer
- Default locale: uk

## Activity Log
- lib/activity-logger.ts: logActivity() helper records create/update/delete/stage_change actions
- ActivityLog model in DB: entity, entityId, action, details, userId, timestamp
- Activity log page at /activity-log with filters by entity/action/date
- Logging integrated into leads, deals, tasks API routes

## Automation Execution
- API endpoint: /api/automations/execute (POST with secret)
- Handles 5 trigger types: task_overdue, no_response, new_lead, deal_closed, stage_change
- Automation model has lastRunAt/lastRunResult fields
- Scheduled daemon runs every 2 hours via POST to the endpoint
- AUTOMATION_SECRET env var for authentication

## Key Technical Details
- Constants in lib/constants.ts (fallback), dynamic stages from FunnelStage table
- Cloud storage (S3) for photos and avatars
- xlsx package for Excel import
- Prisma with PostgreSQL, shared dev/prod DB
- batch_file_write fails with (dashboard) parenthesized dirs — use bash cat > instead
- Presigned upload API returns uploadUrl and cloud_storage_path (not url/cloudStoragePath)
