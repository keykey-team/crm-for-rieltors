## Огляд продукту

### Що таке KeyKey CRM?

**KeyKey CRM** — це комплексна система управління взаємовідносинами з клієнтами (CRM), спеціально розроблена для рієлторів та агентств нерухомості. Система дозволяє автоматизувати продаж нерухомості, управляти лідами, угодами та об'єктами нерухомості в одному місці.

### Цільова аудиторія

| Сегмент | Опис |
|---------|------|
| **Рієлтори-фрілансери** | Один агент, кілька клієнтів, базова функціональність |
| **Дрібні агентства** | 2-10 агентів, керівник, аналітика |
| **Середні агентства** | 10-50 агентів, декілька відділів, автоматизація |
| **Великі мережі** | 50+ агентів, комплексна аналітика, інтеграції |

### Основне значення продукту

✅ **Централізована база** — Всі ліди, угоди, об'єкти в одній системі  
✅ **Автоматизація** — Правила для автоматичних дій та нотифікацій  
✅ **Аналітика** — Детальні звіти про продажі та конверсію  
✅ **Дешево** — Вбудовані інструменти замість купівлі 10 окремих сервісів  
✅ **Локалізовано** — Українська мова, підтримка гривні та UAH рынку

## Основні особливості

### 🎯 Управління лідами

**Що це**: Централізована база всіх потенційних клієнтів

**Функції**:
- Додавання лідів вручну, з Telegram, Instagram, OLX, dom.ria
- Статуси: Новий, Активний, Теплий, Холодний, Втрачений
- Фільтрація за джерелом, статусом, бюджетом, типом нерухомості
- Призначення агентам
- Приватні примітки та історія комунікацій
- Експорт в Excel

**Бізнес-цінність**:
- Не втратяться ніякі контакти
- Агенти бачать повну історію взаємодії
- Керівник контролює розподіл роботи

---

### 🏠 Управління об'єктами нерухомості

**Що це**: Каталог всіх об'єктів на продаж/здачу

**Функції**:
- Додавання об'єктів з описом, характеристиками, фото
- Типи: Квартира, Будинок, Комерція, Земля
- Статуси: Активний, Проданий, Зарезервований, Неактивний
- Галерея фото (завантаження на AWS S3)
- Переклад характеристик за районами
- Посилання угод до об'єктів

**Бізнес-цінність**:
- Одна об'єкт = багато потенційних покупців
- Фото та деталі завжди актуальні
- Знайти аналоги можна за 2 клікування

---

### 📋 Управління угодами (Воронка продажу)

**Що це**: Kanban-дошка для відстеження угод від нового лід до закриття

**Функції**:
- Drag-and-drop між стадіями воронки
- Кастомізація стадій для кожного агентства
- Деталі угоди: контакти, об'єкт, сума, комісія
- Коментарі з @mention
- Чек-лісти (документи, показ, підписання)
- Дати: зустріч, показ, внесок, закриття
- Часова шкала угоди

**Типові стадії**:
1. **Новий лід** → Залучення, уточнення потреб
2. **Кваліфікований** → Готовий до показу
3. **На показі** → Проведена презентація
4. **Переговори** → Узгодження деталей
5. **Угода** → Підписання документів
6. **Закрита** → Успішно завершена

**Бізнес-цінність**:
- Керівник бачить, де застрягла угода
- Прозорість на 100%
- Прогнозування доходу

---

### ✅ Управління задачами

**Що це**: Todo-листи для дзвінків, зустрічей, документів

**Функції**:
- Типи: Дзвінок, Повідомлення, Зустріч, Показ, Документи
- Пріоритети: Низький, Середній, Високий
- Статуси: На очікуванні, В процесі, Готово
- Терміни виконання
- Призначення агентам
- Нотифікацій

**Бізнес-цінність**:
- Ніщо не забудеться
- Точка входу для помічників/новачків

---

### 📅 Календар подій

**Що це**: Синхронізований календар для показів, зустрічей, дедлайнів

**Функції**:
- Місячний та тижневий вид
- Перетягування подій
- Приватні та спільні календарі
- Експорт в ICS для Outlook/Google Calendar
- Нотифікації перед подією
- Інтеграція з Event моделлю

**Бізнес-цінність**:
- Агенти готові до показів вчасно
- Менеджер розподіляє час матеріально
- Синхронізація між системою та особистим календарем

---

### 💬 Внутрішній чат

**Що це**: Месенджер для команди (не для клієнтів)

**Функції**:
- Кімнати для груп/проектів
- Прямі повідомлення
- @mention для привернення уваги
- История з можливістю поиска

**Бізнес-цінність**:
- Командна комунікація в системі
- Не потрібні Telegram/Slack для внутрішніх справ
- Контекст прив'язаний до лідів/угод

---

### 🔄 Автоматизація

**Що це**: Правила "Якщо X, то Y" для автоматичних дій

**Типові сценарії**:
- Новий лід → Відправити вітальне повідомлення
- Лід не активний 7 днів → Зменшити пріоритет
- Угода на показі > 14 днів → Нагадати керівнику
- Комісія розраховується → Розіслати звіти

**Реалізація**:
- Користувач встановлює тригери (trigger)
- Користувач встановлює дії (action)
- Система виконує автоматично

**Бізнес-цінність**:
- Економія часу на рутину
- Консистентність процесів
- Менше помилок

---

### 📖 База знань

**Що це**: Вікі-подібний простір для документації

**Контент**:
- Скрипти для дзвінків
- Чек-листи для процесів
- Юридична інформація
- FAQ для клієнтів

**Бізнес-цінність**:
- Новачки можуть швидко навчатися
- Консистентність розмов з клієнтами
- Знання не в головах

---

### 📊 Аналітика та звіти

**Що це**: Детальні графіки та метрики продажу

**Метрики**:
- Загальна кількість лідів за період
- Конверсія: лід → угода
- Середня сума угоди
- Комісія за період
- Активність по агентам
- Джерела лідів (звідки найбільше)
- Хронологія продажу

**Графіки**:
- Line chart: тренд лідів за часом
- Pie chart: розподіл по джерелам
- Bar chart: топ агентів
- Funnel chart: конверсія на кожній стадії

**Бізнес-цінність**:
- Дані для прийняття рішень
- Мотивація для команди (лідерборди)
- Прогнозування доходу

---

### 📝 Журнал дій (Activity Log)

**Що це**: Аудит-лог для керівників

**Записується**:
- Хто створив лід, угоду, задачу
- Хто змінив статус
- Коли була остання зміна
- Який був старий статус → новий статус

**Бізнес-цінність**:
- Прозорість (хто що зробив)
- Розслідування помилок
- Відповідальність

---

### 🎨 Налаштування та кастомізація

**Що це**: Персоналізація системи під агентство

**Опції**:
- **Назва компанії** та логотип (в рахунках, експортах)
- **Кольір бренду** (#073B34 default, але можна змінити)
- **Воронка продажу** (додавати/видаляти стадії)
- **Тема** (світла/темна/система)
- **Кастомні поля** для угод (для спеціалізованих потреб)
- **Довідники** (райони, типи нерухомості)

**Бізнес-цінність**:
- Система відчувається своєю
- Адаптація під унікальний процес агентства

## Ролі користувачів

### 1. 👨‍💼 **Admin** (Адміністратор)

**Права**:
- Повний доступ до всього
- Управління користувачами (додавання, видалення)
- Налаштування системи та воронки
- Білінг та підписка
- Видалення даних
- Білінг та контроль витрат

**Типовий користувач**:
- Власник агентства або IT людина

---

### 2. 👔 **Director** (Директор/Керівник)

**Права**:
- Перегляд всіх лідів/угод (не тільки своїх)
- Редагування даних інших агентів
- Аналітика та звіти
- Розподіл роботи між агентами
- Редагування воронки
- Не може видаляти користувачів чи змінювати тариф

**Типовий користувач**:
- Керівник відділу продажу
- Старший агент з правами управління

---

### 3. 🚀 **Agent** (Агент)

**Права**:
- Вид тільки своїх лідів/угод (придбаних)
- Редагування своїх даних
- Добавіння/редагування пов'язаних задач
- Базова аналітика (свої метрики)
- Видалення власних запис

**Типовий користувач**:
- Рієлтор
- Агент продажу

## User Journey

### Сценарій 1: Новий агент прес-їде в систему

```
Day 1: Onboarding
├─ Отримує посилання на реєстрацію
├─ Входить в систему
├─ Бачить привіт та туру
├─ Переходить в "База знань" для ознайомлення
└─ Готовий почати роботу

Day 2-3: Перші угоди
├─ Переглядає свій список лідів (0, якщо новий)
├─ Керівник розповів 1-2 лід за розпочав
├─ Додає задачу: "Дзвінок клієнту"
├─ Дзвонить (поза системою)
├─ Оновлює статус ліда на "Активний"
└─ Переводить угоду в "На показі"

Week 1: Повна робота
├─ Використовує календар для планування показів
├─ Додає нові ліди вручну
├─ Отримує нотифікації про задачи
├─ Видит аналітику своїх лідів
└─ Починає розуміти процес
```

### Сценарій 2: Керівник контролює команду

```
Monday Morning:
├─ Переглядає Dashboard
│  ├─ 45 активних лідів
│  ├─ 12 угод в процесі
│  ├─ 3 угоди готові до закриття
│  └─ Графік конверсії
├─ Розповідает командиру на планерці
├─ Переглядает Воронку (Kanban)
│  ├─ Бачить, де застрягли угоди
│  ├─ Говорит агентам "Клієнт ХХХ скоро уходит, ускорьтесь"
│  └─ Перемещает угоду между стадиями
├─ Распределяет новых лидов между агентами
└─ Отправляет мотивационное сообщение в чат

Mid-week:
├─ Глядит Аналитику
│  ├─ Какие источники приносят лучшие лиды
│  ├─ Какой агент самый продуктивный
│  └─ Какой процент конверсии
├─ Корректирует стратегию
├─ Обновляет автоматизацию (если нужно)
└─ Пишет статью в базу знаний

Friday:
├─ Смотрит Журнал дій
│  ├─ Кто что изменял
│  ├─ Заметил что Артем переместил сделку в неправильный статус
│  └─ Напоминает ему в чате
└─ Отправляет еженедельный отчет владельцу
```

### Жизненный цикл угоды

```
┌────────────────────────────────────────────┐
│  Новий лід (привязуємо угоду)             │
├─ leadId = ID ліда                         │
├─ propertyId = ID об'єкту (якщо відомий)   │
├─ amount = сума (орієнтовна)                │
├─ commission = комісія (%)                  │
└────────┬───────────────────────────────────┘
         │
    Стадії (кастомізуємо):
    ├─ new_lead (це угода від новичка)
    ├─ qualified (перемовор прошедшие успешно)
    ├─ proposal_sent (счет отправлен)
    ├─ negotiation (переговоры в процессе)
    ├─ deal_ready (почти закрыто)
    ├─ closed_won (закрыто успешно)
    └─ closed_lost (потеряна сделка)

Важные даты в угоде:
├─ meetingDate: когда встречались с клиентом
├─ showDate: когда показывали объект
├─ depositDate: когда внесли задаток
└─ closeDate: когда закрывалась сделка
```

### Правила приватності даних

1. **Агент видит только свои лиды/сделки** (если он не админ или директор)
2. **Директор видит все**, но не может удалять учетные записи
3. **Админ видит и может менять все**
4. **Данные в журнале дей** (activity log) доступны только директору+админу

## Метрики та KPI

### Основні метрики для агентства

```
1. ЛИДИ
   - Загальна кількість: 200+/місяць
   - Качество: % конверсії = угоди/ліди
   - Джерело: 40% Telegram, 30% OLX, 30% інше

2. УГОДИ
   - Середня сума угоди: $50,000
   - Кількість закритих: 10+/місяць
   - Комісія: $1,500-5,000/угода

3. АГЕНТИ
   - Лідів/агента: 20-50/місяць
   - Угод/агента: 1-3/місяць
   - Мотивація: TOP агент отримує +10% бонус

4. КОНВЕРСІЯ
   - Лід → Контакт: 80%
   - Контакт → Показ: 60%
   - Показ → Угода: 30%
   - Загальна конверсія: 14%

5. ЧАСУ
   - Час від ліда до закриття: 30-60 днів
   - Час без активності (дропаут): > 14 днів
```

| **Переходи в систему** | 1+/день | Нотифікації |
| **Задачи до дедлайну** | 100% | Напомінання |
